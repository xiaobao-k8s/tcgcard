### 2.2 数据准确性

4 条进化链全部通过 `validate-cards.ts` 的 6-8 项检查：
- 引用完整性：所有 evolves_from/evolves_to 引用均指向存在的卡片
- 反向一致性：双向引用正确匹配
- 阶段一致性：stage 1/2/3 与进化关系完全吻合

#### DP 数值验证

- **进化链 DP 递增规律**：4 条进化链的 DP 数值均单调递增，增幅合理（每阶段 +30~40）
- **稀有度梯度**：legendary > ultra-rare > rare > common，梯度正确
- **超梦(155/140)、快龙(150/145)** 传说级最终阶段数值最高，符合定位
- **validate-cards.ts check9**：所有 DP 值均在定义的范围内，0 error

**7 个 DP WARN**：均为 `no-evo` 类型卡片（皮卡丘、胖丁、耿鬼、信使鸟、菊草叶、火球鼠、小锯鳄），这些是不进化单卡，DP 范围表中未定义 `no-evo` 范围。这是验证脚本设计上的小缺口，不影响数据正确性。

#### 精灵信息准确性

所有 23 张卡片的中文名、日文名、全国图鉴编号、身高体重均与官方图鉴数据一致。

#### A1 (Note) -- `电` vs `雷` 属性命名不一致

**问题**：`xfd-010` 皮卡丘的 `attribute` 字段值为 `电`，但 `src/lib/attribute-emoji.ts` 中电属性使用 `雷` 作为 key。皮卡丘在 UI 上将显示默认 emoji `✨` 而非 `⚡`。

**影响**：目前仅影响 1 张卡片（皮卡丘），但如果批量录入时沿用 `电` 命名，将影响所有电属性卡片。

**建议**：在批量录入前统一属性命名规范。推荐在 `ATTRIBUTE_EMOJI` 中添加 `电: "⚡"` 作为别名，因为 `电` 更符合日常用语，且在 gen-prompts.ts 的 `ATTRIBUTE_EFFECTS` 映射中也使用 `电`。
