# 奇多神奇宝贝光栅3D变换卡 · 卡片百科网站设计文档

> 设计日期：2026-07-06
> 状态：已确认，待实现

## 一、项目概述

### 1.1 项目定位

**童年卡片资料百科**——面向所有奇多神奇宝贝卡片收藏爱好者，完整收录两代卡片图鉴信息。核心理念：童年没集齐的，来这里补全。

- 纯展示型网站，无需登录
- 所有卡片资料本地静态数据，开源在 GitHub
- 社区可 PR 贡献卡片图片、数据补充

### 1.2 卡片背景资料

| 代 | 名称 | 年份 | 形状 | 数量 | 光栅效果 |
|---|------|------|------|------|----------|
| 第一代 | 闪变进化·旋风卡 | 2001 | 圆形(≈4cm) | 56张 | 双变：进化前后对比 |
| 第二代 | 进化比斗卡 | 2002 | 方形 | 93张(90精灵+3人物) | 部分三变：常态→蓄力→大招 |

**稀有度梯度**：普通 → 中档稀有 → 顶级怨念稀有（龙系进化链/超梦/梦幻/洛奇亚/凤王/雪拉比）

## 二、技术架构

### 2.1 技术选型

- **框架**：Next.js (App Router)，静态站点生成 (SSG)
- **部署**：GitHub Pages / Vercel
- **样式**：Tailwind CSS + 自定义主题
- **数据**：本地 YAML/JSON 文件，构建时生成静态页面
- **交互**：CSS 3D Transform（光栅翻转模拟）+ localStorage（收集标记）
- **图片**：52poke 精灵立绘参考 + CSS 渲染背面 + 社区贡献模板

### 2.2 为什么不选其他方案

| 方案 | 问题 |
|------|------|
| 数据库/后端 | 数据固定149张，无动态内容，不需要 |
| 外部 API 调用 | 减少依赖，构建时一次性获取即可 |
| SPA (纯 CSR) | SEO 不友好，SSG 对百科类更合适 |

## 三、页面设计

### 3.1 首页 — 卡片总览

**布局**：稀有度气泡网格

- 卡片以**圆形缩略图**展示，呼应一代旋风卡造型
- 圆圈大小按**稀有度缩放**：普通卡最小，神兽/龙系最大
- 稀有卡带**发光阴影**效果，一眼识别
- 顶部搜索栏 + 属性/代际/稀有度筛选栏
- 两代卡片通过 Tab 切换（一代旋风卡 | 二代比斗卡）

**交互**：
- 筛选时圆圈动画过渡（大小变化、位置重排）
- 悬停卡片显示名称 + 编号 tooltip
- 点击进入单卡详情

### 3.2 单卡详情页

**布局**（从上到下）：

1. **面包屑导航**：返回图鉴 / 代际 / 属性
2. **卡片正面区域**：大圆形卡片居中，CSS 3D 模拟光栅翻转
   - 鼠标悬停/移动端倾斜时自动切换进化前后画面
   - 显示当前画面状态标签（"小火龙"/"火恐龙"）
3. **进化链导航**：横排展示完整进化链，当前精灵高亮
4. **卡片背面数据**：
   - 编号、中日文名、属性
   - 身高体重、经典技能
   - DP攻击/防御数值（二代加 DP速度）
   - 图鉴描述
5. **稀有度信息**：稀有度等级 + 当年交换行情描述

### 3.3 进化链页面

- 按进化关系分组（如「迷你龙 → 哈克龙 → 快龙」）
- 每组横排串联展示，箭头连接
- 按属性/世代分区
- 点击任意精灵跳转详情

### 3.4 稀有度榜单

- 按稀有度分档排列：传说级 → 极稀有 → 稀有 → 普通
- 传说级卡片大尺寸置顶展示
- 附带「当年交换行情」描述文案（如"1张快龙 ≈ 20张普通卡"）
- 按出现概率估算展示稀有度星级

### 3.5 对战规则页

- 还原课间 DP 对战规则：
  - 双方各出一张 → 比攻击 → 比防御 → （二代）比速度
  - 三局两胜
- 动画示意对战流程
- 附带实际卡片 DP 数值案例
- 拍卡玩法补充说明

## 五、视觉设计

### 5.1 色彩系统

| 用途 | 色值 | 说明 |
|------|------|------|
| 主色（奇多橙） | `#f97316` | 品牌主色调 |
| 背景暖色 | `#fffbeb` | 温暖米黄底色 |
| 卡片底色 | `#fef7ed` | 卡片容器暖白 |
| 强调色-稀有 | `#ef4444` | 稀有标签/发光 |
| 强调色-传说 | `#8b5cf6` | 传说级发光 |
| 文字主色 | `#78350f` | 深棕 |
| 文字辅色 | `#a16207` | 浅棕 |
| 边框色 | `#fde68a` | 暖金边框 |

### 5.2 设计原则

- 圆形元素为主（呼应一代旋风卡）
- 稀有度越高视觉权重越大
- 暖色系 + 略微怀旧质感
- 光栅效果用 CSS 3D transform 模拟
- 移动端触摸倾斜触发翻转

## 六、卡片图片策略（迭代方案）

### 阶段 1：纯 CSS 占位
- 卡片正面用 CSS 渐变色圆片 + 精灵 emoji/编号占位
- 背面数据纯 CSS 渲染
- 光栅反转用 CSS 3D rotateY/tilt 动画模拟
- 不同属性用不同渐变色区分

### 阶段 2：精灵立绘引入
- 从 52poke 获取精灵官图/立绘作为卡片正面
- 构建时脚本下载压缩
- 保留 CSS 光栅翻转效果

### 阶段 3：社区贡献
- 提供卡片图片规范模板
- GitHub Issue/PR 流程接收图片贡献
- 支持 AI 生图（多模态模型生成卡片风格图）

### 图片规范（给社区贡献者）
- 格式：PNG，透明背景优先
- 圆形卡：400×400px，精灵居中
- 光栅帧：400×400px × 2（进化前/后）
- 命名规则：`{card-id}-front.png`、`{card-id}-frame-a.png`、`{card-id}-frame-b.png`

## 七、非功能需求

- **性能**：Lighthouse 评分 > 90，首屏加载 < 2s
- **响应式**：移动端/平板/桌面三端适配
- **可访问性**：语义化 HTML，图片 alt 文本
- **国际化**：仅中文，精灵名保留日文
- **维护性**：新增卡片只需添加 YAML 文件

## 八、不做的事项（明确排除）

- 用户注册/登录系统
- 在线交易/电商功能
- 后端 API/数据库
- 社区评论/论坛功能
- 实时对战游戏
- 国际化多语言
